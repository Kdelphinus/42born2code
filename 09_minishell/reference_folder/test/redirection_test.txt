리다이렉션 파일 생성 우선순위  체크 
echo hi > a > b > c > d | cat d 할때 hi가 출력 되야함 

echo 123 > 00 99 88 < 00
0이라는 이름의 파일 안에 123 99 88 이 담겨있습니다.
echo 123을  input으로 00에 넣어주고 이후에 00을 input으로  88 쪽으로 넣어주는데 결과가 왜 00에 123 99 88 이 모두 담겨있을까요????


why?
>' 리디렉션은 바로 옆에 있는 문자열만 파일명으로 취급하구 그다음 문자열부턴 echo의 인자로 취급됩니다
'<' 리디렉션은 88에 00을 넣어주는것이 아닌 명령어 echo에 00에 들은 내용물을 출력하게 해주는 리디렉션입니다
88은 파일명이 아니라 echo의 인자인 문자열로 들어가고 있습니다

echo: 명령어
123: echo의 인자
'>' : 출력 리디렉션
00: 출력 리디렉션이 가리키는 파일명 (위의 출력 리디렉션에서는 echo의 처리가 끝나면 그 출력물을 이 파일로 보냅니다)
99: echo의 인자
88: echo의 인자
'<': 입력 리디렉션
00: 입력 리디렉션이 가리키는 파일명


참고로 빌트인 함수들은 출력 리디렉션은 먹히지만 입력 리디렉션은 먹히지 않습니다
따라서 echo < test.txt 의 결과로는 빈 문자열이 출력됩니다
